# Project Governance

> **Project**: JohnPGame
> **Effective Date**: 2026-04-13
> **Status**: Active

This folder contains the governance rules for this project. These rules are **non-negotiable** and apply to all contributors (human and AI).

## Documents

| Document | Purpose | Enforcement |
|----------|---------|-------------|
| [FILE-NAMING.md](FILE-NAMING.md) | File and folder naming conventions | Hard — violating files must be renamed before merge |
| [FOLDER-STRUCTURE.md](FOLDER-STRUCTURE.md) | Approved folder taxonomy and placement rules | Hard — new folders require governance review |
| [VERSION-MANAGEMENT.md](VERSION-MANAGEMENT.md) | Versioning, changelogs, and release rules | Hard — all releases must follow this process |
| [ARCHIVING.md](ARCHIVING.md) | When and how to archive, delete, or deprecate files | Hard — no deletion without archiving review |
| [DOCUMENT-LIFECYCLE.md](DOCUMENT-LIFECYCLE.md) | Document states, review cadence, and ownership | Medium — enforced at review time |
| [ARCHIVE-INDEX.md](../archive/ARCHIVE-INDEX.md) | Index of all archived documents | Auto — updated on every archive operation |

## Compliance

- Every pull request MUST pass governance checks before merge.
- AI agents MUST read this folder before creating new files or folders.
- Violations discovered after merge MUST be fixed in the next session.

## Amendments

Changes to governance documents require:
1. A documented reason in the PR description
2. Approval from the project owner
3. Updated effective date in the changed document
